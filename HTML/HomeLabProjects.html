Before You Start

Prerequisites & System Requirements

Ensure your system meets these requirements before deploying any home lab applications

Hardware

  • 2+ CPU cores (4+ recommended)
  • 4GB+ RAM (8GB+ for multiple apps)
  • 20GB+ free disk space

Operating System

  • Ubuntu 20.04/22.04 LTS
  • Debian 11/12
  • CentOS/RHEL 8+ or Alpine

Network

  • Static IP address
  • Open ports 80, 443
  • Domain name (optional)

SSH Access

  • SSH key authentication
  • Sudo privileges
  • Terminal/SSH client
Problem Solving

Troubleshooting Common Issues

Solutions to frequent problems you might encounter during installation and deployment

"Permission denied" when running docker

Docker requires root privileges or membership in the docker group.

Solution:

sudo usermod -aG docker $USER && newgrp docker

Port already in use error

Another service is using the required port.

Solution:

sudo lsof -i :PORT
sudo systemctl stop SERVICE

Out of disk space

Docker images and volumes consuming too much space.

Solution:

docker system prune -a
docker volume prune

Cannot access app in browser

Firewall blocking the port or wrong IP address.

Solution:

sudo ufw allow PORT/tcp
ip addr show

Container keeps restarting

Configuration error or missing environment variables.

Solution:

docker compose logs -f
docker inspect CONTAINER

Database connection failed

Database container not ready or wrong credentials.

Solution:

depends_on:
- db
# Check env variables

Essential Docker Commands Reference

Start Services

docker compose up -d

View Logs

docker compose logs -f

Stop Services

docker compose down

Restart Service

docker compose restart app

Update Images

docker compose pull && docker compose up -d

List Containers

docker ps -a
